INSPECTION PROCEDURE

1.CHECK PROBLEM SYMPTOM

  1. Perform the procedures indicated by the displayed message and check if the problem symptom recurs.

    Displayed Message
    		Cause
    		Check Item
    "The lines are busy and information was not able to be acquired." is displayed.
    1. DCM malfunction
    2. Excessive number of retries to access
    3. PPP certification error
    4. Error other than PPP certification
    5. No available sockets
    6. No response from the G-BOOK center for a certain period of time after a request is sent
    7. HTTP error
    8. Other communication error
    		Wait for a while, perform the operation again and check if the problem symptom recurs.*
    "The lines are busy and it is not possible to display the desired content. Please try again (operate it again) after waiting for a while." is displayed.
    		Wait for a while, perform the operation again and check if the problem symptom recurs.*
    "The lines are busy. Operate it after waiting for a while." is displayed.
    		Transmission from the DCM is restricted
    		Wait for a while, perform the operation again and check if the problem symptom recurs.*
    "The lines are shut off and information was not able to be acquired." is displayed.
    		Unexpected shutoff of the telephone line
    		Wait for a while, perform the operation again and check if the problem symptom recurs.*
    "Outside the telephone call area. Move to the telephone call area and use it." is displayed.
    1. Vehicle is outside the communication service area
    2. Service becomes unavailable during communication
    		Move the vehicle to a communication service area, perform the operation again and check if the problem symptom recurs.

    HINT:
    *: If this message is temporarily displayed, there is a problem with the radio wave signal conditions or the base station was busy. This is not a malfunction (convince the customer that this is not a malfunction and ask them to use the system as is).
